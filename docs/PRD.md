# 정산도우미 (Settlement Helper) PRD

## 1. 프로젝트 개요

### 1.1 배경
회식이나 모임에서 한 날짜에 여러 차수로 이동하며 결제하는 경우, 차수마다 참석 인원이 달라지거나 술/음료수를 따로 정산해야 하는 상황이 자주 발생한다. 매번 수기로 계산하기 번거롭고 누락이 발생하기 쉬워, 차수별 결제 내역을 입력하면 인원별 최종 정산 금액을 자동으로 계산해주는 도구가 필요하다.

### 1.2 목적
한 모임의 여러 차수 결제 내역을 입력받아, 인원별로 누가 누구에게 얼마를 보내야 하는지 자동 계산하고 표 형태로 출력·공유할 수 있도록 한다.

### 1.3 타겟 사용자
- 회식/모임 정산을 담당하는 총무 또는 결제자
- 차수가 많거나 인원 변동이 잦아 수기 정산이 어려운 경우

---

## 2. 핵심 기능 요약

1. 모임 참석 인원 등록
2. 차수 추가 및 차수별 결제 내역 입력 (전체 분할 / 술·음료 분할)
3. 차수별 참석 인원 변동 관리 (추가/제거/수정)
4. 인원별 정산 결과 표 출력
5. 정산 결과 이미지 다운로드 및 공유

---

## 3. 상세 기능 명세

### 3.1 참석 인원 관리
- 모임에 참석한 인원 명단을 입력할 수 있다.
- 인원 추가/제거/이름 수정이 가능하다.
- 입력된 인원은 모든 차수에서 선택 가능한 풀(pool)이 된다.

### 3.2 차수 관리
- "차수 추가" 버튼으로 새로운 차수를 만들 수 있다.
- 각 차수에는 다음 정보가 포함된다:
  - 차수 이름 또는 번호 (예: "1차 - 삼겹살집")
  - 해당 차수에 참석한 인원 (전체 인원 풀에서 선택)
  - 결제 금액
- 차수는 추가, 제거, 수정이 모두 가능하다.

### 3.3 결제 금액 입력 방식

각 차수는 하나 이상의 **항목(item)** 으로 구성된다. 항목별로 금액과 부담자를 따로 지정할 수 있어, 한 차수 안에서 "공통 비용 + 개별 비용"을 자유롭게 혼합할 수 있다.

**항목 단위 구조**
- 차수 추가 시 기본적으로 항목 1개가 생성된다.
- 사용자는 차수 내에서 "항목 추가" 버튼으로 항목을 계속 추가할 수 있다.
- 각 항목은 다음 정보를 가진다:
  - 항목 이름 (선택, 예: "안주", "술", "음료")
  - 금액
  - 부담자 (해당 차수 참석자 중 다중 선택)
- 각 항목은 그 항목의 부담자들 사이에서 균등 분할된다 (1원 미만 버림).

**사용 예시**

*예시 1 — 단순 균등 분할 (1차)*
- 항목 1: "전체" — 75,000원, 부담자: A, B, C
- 결과: A·B·C 각 25,000원

*예시 2 — 술/음료 분할 (2차)*
- 항목 1: "술" — 45,000원, 부담자: A, B, D
- 항목 2: "음료" — 8,000원, 부담자: C
- 결과: A·B·D 각 15,000원, C는 8,000원

*예시 3 — 공통 + 개별 혼합 (3차)*
- 항목 1: "안주" — 30,000원, 부담자: A, B, C, D (공통)
- 항목 2: "술" — 40,000원, 부담자: A, B (술 마신 사람)
- 항목 3: "음료" — 6,000원, 부담자: C, D (음료 마신 사람)
- 결과: A·B 각 27,500원 (안주 7,500 + 술 20,000), C·D 각 10,500원 (안주 7,500 + 음료 3,000)

**UI 단순화 제안**
- 항목이 1개뿐이고 부담자가 차수 참석자 전원이면, 화면에서는 "균등 분할"로 단순하게 표시한다.
- 항목이 2개 이상이거나 부담자가 일부일 때만 항목 단위 UI가 펼쳐진다.
- 이를 통해 단순한 케이스(예시 1)에서도 입력이 복잡해 보이지 않도록 한다.

### 3.4 정산 결과 출력
다음 항목을 포함한 표를 화면에 표시한다.

| 항목 | 설명 |
|------|------|
| 행(row) | 참석 인원 |
| 열(column) | 차수별 부담액 + 합계 |
| 마지막 행 | 차수별 총 결제액 |
| 마지막 열 | 인원별 최종 정산 금액 |

차수 안에 여러 항목이 있을 경우, 표에서는 인원별로 해당 차수의 부담액 합계만 보여준다 (항목별 상세 분해는 입력 화면에서만 확인). 표가 가로로 너무 길어지지 않도록 하기 위함이다.

예시 (1차: 균등분할 75,000원 / 2차: 안주+술+음료 혼합):

| 인원 | 1차 | 2차 | 합계 |
|------|-----|-----|------|
| A | 25,000 | 27,500 | 52,500 |
| B | 25,000 | 27,500 | 52,500 |
| C | 25,000 | 10,500 | 35,500 |
| D | - | 10,500 | 10,500 |
| **차수 합계** | 75,000 | 76,000 | 151,000 |

### 3.5 결과 다운로드 및 공유
- 정산 결과를 이미지(PNG 또는 JPG) 파일로 다운로드할 수 있다.
- 다운로드한 이미지는 카카오톡 등으로 바로 공유할 수 있는 형식이어야 한다.
- 모바일 환경에서도 다운로드/공유가 정상 동작해야 한다.
- **v1: 표(table) 형태로만 제공**한다. 인원이 행, 차수가 열, 마지막 행에 차수별 합계, 마지막 열에 인원별 최종 정산 금액이 표시된다.
- v2에서 카드 스타일을 추가하고, 사용자가 토글로 표/카드 스타일을 선택할 수 있도록 한다 (섹션 7 참고).

## 4. 사용자 흐름 (User Flow)

1. 앱 진입
2. 참석 인원 입력 (예: A, B, C, D)
3. "1차 추가" → 참석자 선택 (A, B, C) → 항목 1 금액 75,000 입력 → 부담자 전원(A·B·C) 선택
4. "2차 추가" → 참석자 선택 (A, B, C, D)
   - "항목 추가"를 눌러 항목 3개 입력
   - 항목 1 "안주" 30,000원, 부담자: A·B·C·D
   - 항목 2 "술" 40,000원, 부담자: A·B
   - 항목 3 "음료" 6,000원, 부담자: C·D
5. 정산 결과 표 자동 생성 및 화면 표시
6. "이미지로 저장" 버튼 클릭 → 이미지 파일 다운로드
7. 카카오톡 등으로 공유

## 5. 비기능 요구사항

### 5.1 사용성
- 모바일 우선(Mobile-first) 디자인. 회식 직후 폰으로 빠르게 정산하는 시나리오가 주.
- 입력 단계가 직관적이어야 하며, 한 화면에서 차수를 계속 추가할 수 있어야 한다.

### 5.2 데이터
- 별도 회원가입/로그인 없이 사용 가능.
- 작성 중인 정산 내역은 **브라우저 세션 동안만 유지**한다. 탭/브라우저를 닫으면 데이터는 사라진다 (sessionStorage 사용).
- 새로고침(F5) 시에는 데이터가 유지된다.
- 명시적으로 "초기화" 버튼을 눌러 현재 작성 중인 데이터를 비울 수 있다.

### 5.3 정확성
- 1원 단위까지 분할하며, **1원 미만 잔액은 버림(floor) 처리**한다.
- 예: 10,000원 ÷ 3명 = 3,333원씩 부담 (잔액 1원은 버림).
- 결과적으로 인원별 부담액 합계가 실제 결제 금액보다 약간 적을 수 있으며, 이는 의도된 동작이다.
